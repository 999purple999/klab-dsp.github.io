BINAURA — Feature Breakdown

Closed source. Professional wavetable synthesis engine designed for quality, performance and expressivity.

4-Engine Architecture

Four independent lock-free wavetable oscillator engines running simultaneously with per-engine detuning, pan, phase and octave control.

AVX2 SIMD Optimization

CPU-level vectorized processing using AVX2 intrinsics for maximum throughput at any polyphony level with minimal CPU impact.

PolyBLEP Antialiasing

Polynomial band-limited step function applied across all oscillator waveforms to eliminate aliasing artifacts even at extreme high frequencies.

Analog Drift DDS

Per-voice analog instability simulation via direct digital synthesis jitter — emulates the natural imperfection of hardware VCOs.

ZDF Binaural Filter

Zero-delay feedback biquad filter architecture with SSI2144 analog circuit emulation. Self-oscillation, key tracking, velocity modulation.

8-Slot Mod Matrix

Full modulation matrix with 8 assignable slots. Route any source (LFO, env, velocity, aftertouch, MPE) to any destination parameter.

4 LFO Engines

Four independent low-frequency oscillators with sync, tempo lock, per-voice mode, phase randomization and multiple waveform shapes.

MPE + CC74 Support

Full MIDI Polyphonic Expression with per-note pitch bend, slide (CC74), pressure, and glide — designed for Roli, Expressive E and similar controllers.

ThermalGuard DSP

Real-time CPU thermal monitoring with automatic voice limiting and processing fallback to prevent audio dropouts under sustained load.

Oversampling 1x / 2x / 4x

Selectable oversampling ratio per instance. 4x oversampling drastically reduces nonlinear distortion and aliasing in complex patches.

60 Factory Presets

60 professionally crafted presets spanning pads, basses, leads, plucks and evolving textures. Organized by category with tagging system.

Serum-Level UX

Precision-designed JUCE interface with GPU-accelerated waveform rendering, drag-to-assign mod routing and real-time oscilloscope display.

Secure Software

K-Perception — Feature Breakdown

Closed source. Windows & Android only — macOS, iOS and Linux support coming in a future release.

AES-256-GCM Encryption

Every note encrypted at rest and in transit using AES-256 in Galois/Counter Mode. Authenticated encryption prevents tampering at the byte level.

PBKDF2-SHA256 KDF

Master password stretched through PBKDF2 with SHA-256 and a high iteration count. Brute-force resistance built into the key derivation layer.

Biometric Unlock

Fingerprint and face unlock via platform biometric APIs. Vault key stored in secure hardware enclave — never in user-accessible storage.

Drive Sync via PKCE OAuth

Cloud backup and sync through Google Drive using PKCE authorization code flow. Zero plaintext ever leaves the device before encryption.

Offline-First Architecture

All features work fully offline. Sync is additive and conflict-resolved — the vault is always available regardless of network state.

Revision History

Full version history per note with timestamp and diff view. Restore any previous version of any note at any time without data loss.

Encrypted Full-Text Search

Fast full-text search across all notes without ever decrypting to disk. Search index built in-memory from the decrypted working set only.

Note Linking

Wiki-style bidirectional note linking. Build connected knowledge graphs entirely inside your encrypted vault.

Windows & Android

Native applications for Windows desktop and Android mobile. macOS, iOS and Linux are planned and will ship in a future major release.

Zero-Knowledge Architecture

The server never sees your plaintext. Encryption and decryption happen exclusively on-device before any data touches the network or cloud storage.

Multi-Vault Support

Create separate isolated vaults for different contexts — personal, work, projects. Each vault has its own encryption key and unlock credentials.

Tamper Detection

GCM authentication tag validates every decryption operation. Any modification to ciphertext — even a single bit — is detected and rejected before display.
