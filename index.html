Security Architecture

Zero-knowledge by design.
Not by marketing.

Every architectural decision passes one test: if the server and its databases were subpoenaed or breached, could plaintext content be recovered? The answer is no.

AES-256-GCM
Every note encrypted with AES in Galois/Counter Mode. 256-bit key provides 2²⁵⁶ brute-force space. GCM authentication tag detects any modification — even a single bit flip is rejected.
PBKDF2-SHA256 · 310,000 iter
Master password stretched via 310k rounds of PBKDF2-SHA256. NIST SP 800-132 compliant. Unique per-user salt. Brute-forcing costs millions of operations per password attempt.
PKCE OAuth · Google Identity
Login uses OAuth 2.0 PKCE — Proof Key for Code Exchange. Proof key is SHA-256 hashed client-side; only the hash travels to the server. Login is identity only, never the vault key.
Biometric Enclave
Android Keystore backs biometric auth. The vault session key is stored in hardware-isolated TEE. Not accessible to user storage, backups, or ADB extraction.
Share Link Cryptography
Share keys use a random 256-bit key delivered via URL fragment — fragment never reaches the server. Password-protected links add PBKDF2 key stretching on top. Revocation is instant and server-side.
Zero Server Plaintext
Cloudflare D1 + R2 store: change IDs, entity type names, byte counts, timestamps, user IDs. Never: note titles, bodies, tag names, filenames, property values, search queries, or any content.
