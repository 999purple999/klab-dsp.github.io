# Exports

## What it is

You can export any note from K-Perception in multiple formats. Exports produce a decrypted, portable copy of your note — the exported file does not contain any encryption layer from the vault (unless you choose the KPX encrypted portable format). Exports are processed entirely on-device; no plaintext is sent to the server.

## When to use it

- Share a note with someone who does not have K-Perception.
- Archive a document in a format independent of K-Perception.
- Submit a report or paper in PDF format.
- Back up a specific note as a local file.
- Move a vault to a new device before cloud sync is available (use KPX format).

## Supported export formats

| Format | Extension | Description | Available from |
|---|---|---|---|
| Plain text | `.txt` | Raw text content, no formatting | All modes |
| Markdown | `.md` | Markdown source (Markdown mode); best-effort conversion from other modes | All modes |
| JSON | `.json` | Full note structure with metadata | All modes |
| HTML | `.html` | Rendered HTML document with embedded CSS | Docs, Editorial, Markdown |
| PDF | `.pdf` | Print-ready PDF via pdf-lib 1.17.1 | All modes |
| KPX | `.kpx` | Encrypted portable K-Perception export | All modes |

## Step by step

### Desktop

1. With the note open, click the **Export** button in the toolbar (↑ icon) or press `Ctrl+Shift+E`.
2. The export dialog opens. Select the **Format** from the dropdown.
3. For PDF/HTML exports, configure options:
   - **Paper size:** A4, Letter, Legal, or custom dimensions.
   - **Margins:** top/right/bottom/left in mm.
   - **Font family** and **font size**.
   - **Header / footer** text (can include `{title}`, `{date}`, `{page}`, `{total}`).
   - **Theme:** choose from available PDF themes (e.g. clean, night, academic).
4. Optionally **name and save** the current settings as a **named export profile** for re-use.
5. Click **Export** and choose a save location in the OS file dialog.

### Android

1. Open the note.
2. Tap the three-dot menu (⋮) → **Export**.
3. Choose format.
4. The exported file is saved to the Downloads folder. A notification appears with an option to open or share the file.
5. The **Export history** screen (accessible from Settings → Export history) shows all past exports.

## Format details

### PDF export

Generated by **pdf-lib 1.17.1**. The layout respects the note's content structure. In Editorial mode, the PDF honours the page geometry set by CSS variables (margins, page width, font size). Heading styles, code blocks (monospace), tables, and images are all rendered. Callouts render as bordered boxes. Task lists render with check marks. MathML/KaTeX renders via the KaTeX auto-render script loaded in the export HTML; fonts are fetched from the KaTeX CDN when an internet connection is available.

### HTML export

A complete HTML5 document with CSS embedded inline. Suitable for email or static web hosting. Images are base64-encoded inline. Scripts are not included. Mermaid diagrams render as SVG. KaTeX math renders as HTML+CSS (no external dependencies).

### KPX export

A ZIP archive with a JSON manifest and individual encrypted blob files. Each blob is re-encrypted with a KPX-specific AES-256-GCM key derived from a user-chosen export password (PBKDF2-SHA256, 310,000 iterations, 32-byte random salt). The export password is separate from the vault password. See [KPX format](../sync-and-storage/kpx-format.md) for the byte-level specification.

### Named export profiles

A named export profile saves: format, paper size, margins, font, header/footer template, and theme. Profiles are stored in the vault and sync across devices. You can create unlimited profiles.

## Behaviour and edge cases

- **Mode-specific fidelity:**
  - Exporting a **Sheets** note to PDF renders the visible grid. The export pipeline converts the note body to HTML and then prints via Electron `printToPDF`; only the currently active sheet is included in the output.
  - Exporting a **Canvas** note to PDF renders the canvas as a vector SVG embedded in the PDF.
  - Exporting a **LaTeX** note to PDF re-renders the latex.js output — the result matches the preview.
- **Attachments in exports:** Attachments are NOT included in text/Markdown/JSON exports. In PDF/HTML exports, embedded images are included inline. Other attachment types are referenced by filename only.
- **Character encoding:** All exports use UTF-8.
- **Workspace notes:** Workspace document exports work the same way. The export is decrypted on-device — the exported file has no workspace-specific encryption.
- **Batch export:** Batch export of multiple notes is not currently supported [NEEDS-IMPL-DECISION].

## Plan availability

- **Plain text, Markdown, JSON exports:** All plans including Local.
- **PDF export:** Available on all plans. On desktop, PDF export uses Electron's `printToPDF` (no external font service required). On web/Android, PDF rendering fetches KaTeX font assets from CDN when online.
- **HTML export:** All plans.
- **KPX export:** All plans.

## Permissions and roles

Any user who can read a note can export it. Exporting a workspace document does not require admin permission.

## Security implications

Exports are plaintext (except KPX). The exported file is not encrypted by K-Perception (except KPX). You are responsible for the security of the exported file. If you share an exported PDF, the recipient has access to the plaintext content. KPX exports are encrypted — only someone with the export password can read them. Export operations do not generate audit log entries — the export IPC handlers in the main process write files to disk without calling the audit pipeline.
