# Notifications

K-Perception delivers workspace and collaboration notifications through an in-app notification bell. System-level (OS) push notifications are not confirmed in the current source; contact support@kperception.com for current platform notification support.

## Notification Types

Workspace notifications are generated by server-side events. The following event types are supported:

| Event | Description |
|---|---|
| `member_joined` | A new member accepted an invitation and joined the workspace |
| `message` | A channel message was posted |
| `file_uploaded` | A file was uploaded to a workspace directory |
| `file_shared` | A file was shared with you |
| `comment` | Someone commented on a note or document |
| `collab_invite` | You were invited to a collaborative editing session |
| `mention` | You were @-mentioned in a channel or document |
| `version_restored` | A file version was restored |

## In-App Notification Bell

The notification bell appears in the workspace shell header. It displays an unread count badge when new notifications arrive. Clicking it opens the notification drawer, which lists recent events in reverse chronological order.

Notifications are fetched from the server and displayed client-side. They are associated with the current account session and are visible on all signed-in devices after the next sync/refresh.

## System Notifications (OS-Level)

System push notifications (desktop OS toast / Android push) have not been confirmed in the current source code. The following behavior is expected but may vary by platform:

- Desktop (Windows/macOS): May display OS toast notifications for workspace events when the app is running.
- Android: May use Android notification channels for workspace events.
- Web PWA: Browser notification permission may be requested.

## Notification Preferences

Per-notification-type preferences (mute, per-workspace toggle, quiet hours) may be available; this behavior may vary by platform. Check Settings → Notifications or contact support@kperception.com for current details.

## Inbox and Search

The workspace includes an **Inbox** surface that aggregates all notification-level events across workspaces. This is accessible from the workspace navigation and powered by real API endpoints.

## Platform Notes

- **Desktop:** Notifications are visible while the app window is open. Background delivery behavior may vary; verify with your deployment.
- **Android:** Notifications appear in the in-app bell when the app is in the foreground. Background notification delivery depends on Android push infrastructure configuration.
- **Web PWA:** Notifications are fetched when the tab is active. Browser push notification support may vary by browser.

## Security Note

Notification payloads from the server contain metadata (who did what, when) but not note content. Note content is always E2EE and is never included in notification data sent over the wire.
